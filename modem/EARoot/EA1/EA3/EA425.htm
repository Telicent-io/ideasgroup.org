Rather than invent a new way to model the presentation of <i>ModelElements</i>, MODEM simply uses the W3C SVG 1.1 Standard. SVG (Structured Vector Graphics) is a simple XML format for 2D vector graphics.<br/><br/>The <i>SvgCanvas </i>element specifies the canvas information (height, width, etc.) and is equivalent to the #lt;svg#gt; tag. The exemplar attribute on <i>SvgElement </i>records the empty #lt;svg#gt; tag and all its attributes. <br/><br/>For every graphical element on the canvas there will be an <i>SvgElement </i>or <i>SvgElementGroup</i>. As with the SvgCanvas these encode either a #lt;g#gt; tag (for the group) or the appropriate element tag (e.g. #lt;ellipse#gt;, #lt;circle#gt;, #lt;rect#gt;, etc.)<br/><br/>This means the structure of the SVG xml document is broken down into parts. By tracing the whole-part relationships between the <i>SvgCanvas</i>, <i>SvgElementGroup </i>and <i>SvgElement</i>, the entire SVG document can be reconstructed.<br/>
